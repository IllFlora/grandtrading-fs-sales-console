# Grand Trading FS Sales Console

FSはGoogleスプレッドシートを編集せず、GitHub Pages上のWebアプリだけで訪問先の選定・予定登録・活動結果・商談フェーズを更新します。管理者はGoogleスプレッドシートでリード追加と全体サマリーを確認します。

- Spreadsheet ID: `1F84uPktW-sY8Sf3riBHxUw_NZAymTGHOMEI_7DgfVnE`
- マスターアカウント: `neosanepix@gmail.com`

## シート構成

- `00_管理者サマリー`: 営業ステータス、エリア進捗、担当者別活動、直近活動を自動集計
- `01_リードマスター`: 企業情報と現在の営業状態を保持
- `02_活動履歴`: Webアプリからの予定・訪問・商談結果を追記
- `03_設定`: WebアプリへログインできるGoogleアカウント、表示名、権限、状態を管理
- `04_追加リスト取込`: 管理者が新規リードを追加するための取込テンプレート

## FSユーザーの追加

`03_設定` の F:I に、Googleアカウント・表示名・権限（`FS` / `管理者`）・状態（`有効` / `無効`）を1行追加します。有効ユーザーは最大50名まで管理者サマリーへ自動表示され、表示名はWebアプリの担当者名と活動履歴にも使われます。

重要: 設定表への追加だけではGoogle側の権限は付与されません。各FSアカウントには次の両方が必要です。

- 対象Googleスプレッドシートの編集者共有
- OAuth同意画面がテスト公開中の場合は「テストユーザー」への追加（本番公開済みなら不要）

スプレッドシートを共有しても、FSにシートを直接触らせる運用にはしません。Sheets API経由でWebアプリが読み書きするために権限が必要です。

## FSの基本操作

1. Googleアカウントでログイン
2. 「行き先を探す」で候補を絞る
3. 訪問予定日、訪問順、事前メモを登録
4. 「今日の訪問」でルート順に確認
5. 現地で訪問・商談結果、次回対応日、メモを登録
6. 「商談管理」で進行中案件を確認

活動日時はGoogle Sheetのタイムゾーン `Asia/Tokyo` で保存・集計されます。

## 管理者機能

- `neosanepix@gmail.com` はマスターとして管理者設定を編集可能
- 管理者設定からFSユーザーの表示名・権限・状態を更新可能
- 各企業の詳細画面から、その企業のテスト活動を削除して未着手・未割当に戻せる
- 管理者サマリーで担当者ごとの本日活動、直近7日、担当リード、各フェーズ、最終活動を確認可能

## リード追加

管理者が `04_追加リスト取込` に新規リードを追記し、リードマスターへ反映します。既存の活動履歴・担当者設定とは別シートのため、FSユーザー追加やリード追記で過去データは上書きされません。

## 確認済み

- Googleログイン後のセッション復元
- ブラウザの戻る操作で、詳細→一覧→今日の訪問へ復帰
- 訪問予定と訪問結果の登録
- 日本時間の活動日時と管理者サマリーへの即時反映
- FSユーザー追加時の担当者別サマリー自動展開
- 管理者によるテストデータ初期化

このリポジトリにはGoogleのアクセストークンや営業リスト本体は保存していません。データはログイン後にGoogle Sheets APIから取得します。
