FSはGoogleスプレッドシートを編集せず、GitHub Pages上のWebアプリだけで訪問先の選定・予定登録・活動結果・商談フェーズを更新します。管理者はGoogleスプレッドシートでリード追加と全体サマリーを確認します。

## シート構成

- `00_管理者サマリー`: 営業ステータス、エリア進捗、担当者別活動、直近活動を自動集計
- `01_リードマスター`: 企業情報と現在の営業状態を保持
- `02_活動履歴`: Webアプリからの予定・訪問・商談結果を追記
- `03_設定`: WebアプリへログインできるGoogleアカウント、表示名、権限、状態を管理
- `04_追加リスト取込`: 管理者が新規リードを追加するための取込テンプレート
- `05_FS個人別サマリー`: FSごとの活動量、対応企業、訪問、有効接触、商談化、成約、転換率、現在フェーズを自動集計
- `06_FS日次実績`: 有効な全FSについて直近90日の日別活動・成果率を自動展開

## FSユーザーの追加

`03_設定` の F:I に、Googleアカウント・表示名・権限（`FS` / `管理者`）・状態（`有効` / `無効`）を1行追加します。有効ユーザーは最大50名まで各サマリーへ自動表示され、表示名はWebアプリの担当者名と活動履歴にも使われます。

Apps Script `GrandTrading FS自動共有` が5分ごとに設定表を確認し、有効ユーザーへスプレッドシートの編集者権限を自動付与します。無効へ変更したユーザーは自動共有対象から外れます。追加後、反映まで最大5分待ってからWebアプリへログインしてください。

Google OAuthは外部ユーザー向けの本番環境へ公開済みです。初回ログイン時のみ「このアプリはGoogleで確認されていません」が表示されるため、「詳細」から続行し、Google Sheets権限を許可します。

スプレッドシートを共有しても、FSにシートを直接触らせる運用にはしません。Sheets API経由でWebアプリが読み書きするために権限が必要です。

## FSの基本操作

1. Googleアカウントでログイン
2. 「行き先を探す」で候補を絞る
3. 訪問予定日、訪問順、事前メモを登録
4. 「今日の訪問」でルート順に確認
5. 現地で訪問・商談結果、次回対応日、メモを登録
6. 「商談管理」で進行中案件を確認

活動日時はGoogle Sheetのタイムゾーン `Asia/Tokyo` で保存・集計されます。

## 管理者機能

- `neosanepix@gmail.com` はマスターとして管理者設定を編集可能
- 管理者設定からFSユーザーの表示名・権限・状態を更新可能

- 管理者サマリーで担当者ごとの本日活動、直近7日、担当リード、各フェーズ、最終活動を確認可能
- 個人別サマリーで訪問→有効接触→商談化→成約の歩留まりと現在フェーズを確認可能
- 日次実績で営業件数に対する商談率・成果率を日別、FS別に確認可能

## リード追加

管理者が `04_追加リスト取込` に新規リードを追記し、リードマスターへ反映します。既存の活動履歴・担当者設定とは別シートのため、FSユーザー追加やリード追記で過去データは上書きされません。
